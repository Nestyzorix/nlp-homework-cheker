# NLP Homework Checker

Интеллектуальная система для проверки домашних заданий на естественных языках.
Проект разработан в рамках ВКР. Система позволяет преподавателю создавать
задания с эталонным ответом и ключевыми словами, а студенту отправлять текстовый
ответ и получать автоматическую оценку с обратной связью.

## Что делает система

- регистрирует пользователей с ролями `teacher` и `student`;
- авторизует пользователей через JWT-токен;
- дает преподавателю создавать задания;
- показывает студенту доступные задания;
- принимает ответ студента;
- сравнивает ответ с эталоном по семантическому сходству;
- проверяет наличие ключевых смысловых элементов;
- применяет простые корректирующие эвристики;
- сохраняет оценку и обратную связь;
- показывает преподавателю ответы студентов и результаты проверки;
- показывает студенту его оценки.

## Подготовка базы данных

Проект ожидает PostgreSQL с базой данных:

```text
HomeworkChecker
```

Текущая строка подключения находится в `app/database.py`:

```text
postgresql://postgres:postgres@localhost:5432/HomeworkChecker
```

То есть по умолчанию используются:

- пользователь: `postgres`;
- пароль: `postgres`;
- host: `localhost`;
- порт: `5432`;
- база: `HomeworkChecker`.

Если параметры PostgreSQL отличаются, нужно изменить `DATABASE_URL` в
`app/database.py`.

## Запуск backend

Из корня проекта:

```bash
venv\Scripts\activate
uvicorn app.main:app --reload
```

Backend будет доступен по адресу:

```text
http://127.0.0.1:8000
```

Документация FastAPI:

```text
http://127.0.0.1:8000/docs
```

## Запуск frontend

В отдельном терминале:

```bash
cd frontend
npm install
npm run dev
```

Frontend будет доступен по адресу, который покажет Vite. Обычно это:

```text
http://localhost:5173
```

В `frontend/package.json` команды запуска сейчас явно используют Node.js из
`C:\Progra~1\nodejs\node.exe`, потому что на компьютере может находиться старая
версия Node.js в другом месте PATH.

## Основные API-маршруты

Auth:

- `POST /register` - регистрация пользователя;
- `POST /login` - вход и получение JWT-токена.

Задания:

- `POST /assignments` - создание задания преподавателем;
- `GET /assignments` - список заданий;
- `GET /assignments/{assignment_id}` - получение одного задания;
- `DELETE /assignments/{assignment_id}` - скрытие задания от студентов.

Ответы:

- `POST /submit_answer` - отправка ответа студентом;
- `GET /results` - просмотр результатов преподавателем;
- `GET /my_results` - просмотр своих результатов студентом.

Диагностика алгоритма:

- `POST /check` - проверка ответа без сохранения результата.

## Как работает алгоритм оценки

Основная логика находится в `app/services/nlp_service.py`.

Алгоритм использует два основных показателя:

- `semantic_score` - семантическая близость ответа студента к эталонному ответу;
- `keyword_score` - полнота найденных ключевых слов и смысловых элементов.

После этого рассчитывается итоговый балл, применяются эвристические штрафы и
балл переводится в оценку от 2 до 5.

Текущая реализация является базовой и будет дорабатываться: планируется добавить
настраиваемые профили оценивания, сохранять параметры проверки и подготовить
экспериментальный модуль для обоснования весов и порогов в ВКР.

## Роли пользователей

`teacher`:

- создает задания;
- смотрит созданные задания;
- скрывает задания от студентов;
- просматривает ответы студентов и результаты проверки.

`student`:

- смотрит доступные задания;
- отправляет ответы;
- просматривает свои оценки и обратную связь.

## Важные замечания

- Это учебный проект для ВКР, не production-система.
- `SECRET_KEY` сейчас хранится прямо в `app/auth.py`; для production его нужно
  вынести в переменные окружения.
- Миграции Alembic пока не настроены. Существующие дополнительные поля базы
  добавляются через SQL-команды при старте приложения.
- Система рассчитана на задания с относительно однозначным эталонным ответом, а
  не на сочинения и свободные дискуссионные ответы.
